generateVerificationCode() → {string}

Generates a cryptographically-weak but sufficiently random 6-digit code.

Algorithm: Math.floor(100000 + Math.random() * 900000) guarantees the result is always in the range [100000, 999999] (6 digits, no leading zero). For higher security requirements consider crypto.randomInt.

(async) hasPendingVerification(email) → {Promise.<boolean>}

Checks whether there is an active (non-expired, non-used) verification code for the given email.

Used by the registration flow to prevent sending multiple codes in quick succession, which would confuse users and waste email quota.

(async) verifyCode(email, code) → {Promise.<{isValid: boolean, message: string}>}

Validates a verification code submitted by the user.

The code is valid only if it exists in the database, has not been used, and has not yet expired. On successful validation the code is marked as used so it cannot be replayed.
