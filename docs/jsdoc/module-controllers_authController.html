Module: controllers/authController

Authentication controller — handles all auth-related HTTP requests.

Implements the two-step email-verification registration flow, local email/ password login, and OAuth login via Google and Facebook.

Registration flow (two-step):

  1. POST /api/auth/register/initiate → initiateRegistration Validates input, checks for duplicates, generates a 6-digit OTP via module:services/verificationService and sends it by email.
  2. POST /api/auth/register/verify → verifyEmailAndRegister Validates the OTP, hashes the password with bcrypt (cost 12), inserts the user row and returns a JWT.

Social login flow:

  • Google: access token is exchanged with Google's userinfo endpoint to obtain the canonical Google user ID. A user row is created on first login or linked to an existing row that shares the same email.
  • Facebook: token is debugged via the Graph API debug_token endpoint before the user record is fetched, ensuring the token was issued for this application.

(static, constant) facebookAuth :function

Authenticates or registers a user via a Facebook OAuth access token.

Uses the debug_token endpoint to validate the token against this application's FACEBOOK_APP_ID, preventing token injection from other Facebook apps. User data is then fetched from the Graph API.

Edge case: if Facebook does not expose the user's email (a privacy setting on the account), a placeholder address fb_<facebook_id>@placeholder.facebook is used so the user_email NOT NULL constraint in the database can be satisfied.

(static, constant) googleAuth :function

Authenticates or registers a user via a Google OAuth2 access token.

The token is verified by calling Google's userinfo endpoint. An existing user is looked up first by google_id, then by email (to link accounts created before OAuth was added). New users are auto-registered.
